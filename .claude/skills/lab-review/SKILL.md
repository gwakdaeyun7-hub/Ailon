# Lab Review -- 학문원리 콘텐츠 교육 효과 검토 및 수정

LAB 탭의 curated 학문원리 콘텐츠(`scripts/curated_principles/*.md`)가 사용자의 **학문 원리 이해에 실제로 도움이 되는지** 검토하고, Gold Standard(SA)와 동등한 교육적 퀄리티로 수정합니다.

> **principle-qa와의 차이**: `principle-qa`는 구조/형식/길이 등 기계적 기준을 검사합니다.
> `lab-review`는 **교육적 효과** — 읽었을 때 원리가 머릿속에 그려지는가, 시각적으로 이해 가능한가 — 에 집중합니다.

## 사용법

### 기본 명령

- `/lab-review` -- 미검토 원리 자동 선택 (1개, 우선순위 기반)
- `/lab-review opt_gradient_descent` -- 특정 원리 검토
- `/lab-review all` -- 전체 일괄 검토 (리포트만)
- `/lab-review fix neuro_attention_mechanism` -- 검토 없이 바로 수정

### 추가 명령

- `/lab-review scan` -- 빠른 스캔: 전체 45개 렌더링 호환성 + 길이 + 구조만 체크 (에이전트 없이 1분 내)
- `/lab-review compare {seed_id}` -- SA와 나란히 비교: 5개 차원별 구조 대조
- `/lab-review dashboard` -- 검토 현황: 등급 분포, 미검토 목록, 우선순위 큐
- `/lab-review sim-audit` -- 시뮬레이션 필요성 일괄 평가 (41개 미보유 원리)

## Gold Standard: Simulated Annealing이 잘 가르치는 이유

**참조 파일:** `scripts/curated_principles/opt_simulated_annealing.md`

SA가 교육적으로 우수한 이유를 다음 5가지로 분석한다. 모든 원리는 이 5가지를 갖춰야 한다.

### 1. 내러티브 아크 (원래 학문 → AI)

SA는 "금속 가열 → 서서히 냉각 → 안정한 결정 구조" 라는 **물리적 이야기**로 시작한다. 독자가 원래 학문의 직관을 먼저 잡은 후에야 알고리즘으로 넘어간다. 이것이 핵심이다:

```
[좋은 예 - SA]
물리적 현상 (금속 열처리)
  → 핵심 인사이트 ("장벽을 넘는 자유")
    → 역사 (Metropolis 1953 → Kirkpatrick 1983)
      → 대응 관계 (온도→유연성, 에너지→목적함수)
        → 메커니즘 (Metropolis 알고리즘 5단계)
          → 현대 AI 연결
            → 한계

[나쁜 예]
"SA는 최적화 알고리즘이다" → 수식 → "물리에서 영감" (한 줄) → 끝
```

**체크포인트:**
- 첫 섹션이 원래 학문 맥락에서 독립적으로 이해 가능한가?
- "핵심은 이것이다" 수준의 insight 문장이 있는가?
- 물리/생물/수학적 직관이 알고리즘 설명보다 먼저 나오는가?

### 2. 공간적 비유 (추상 → 시각)

SA의 가장 강력한 교육 장치: "목적 함수의 값을 높이로 표현한 울퉁불퉁한 산악 지형에서, 가장 깊은 골짜기를 찾는 것이 목표다." 이 한 문장으로 독자는 SA를 **머릿속에 그릴 수 있다**.

**체크포인트:**
- 핵심 개념마다 공간적/시각적 비유가 하나 이상 있는가?
- 비유가 정확한가? (오해를 유발하지 않는가?)
- 비유만으로 핵심 메커니즘의 60% 이상 이해 가능한가?
- 콘텐츠가 과도하게 길지 않은가? (시각적 피로 없이 스크롤 가능한 분량)

### 3. 수식의 교육적 해체

SA의 수식 `P = e^(-dE/T)` 는 단순히 제시되지 않는다:
- 각 변수(dE, T)의 물리적 의미 설명
- "T가 높을수록, dE가 작을수록 수용 확률이 올라간다" — 변수 변화의 효과
- 볼츠만 상수 k가 생략된 이유 — 물리학과의 차이
- 물리적 원리가 어떤 충실도로 보존되었는지

**체크포인트:**
- 수식이 있으면 각 변수의 의미가 풀어서 설명되어 있는가?
- 수식의 직관적 해석(어떤 값이 커지면 어떻게 되는가)이 있는가?
- SnapsContentRenderer 수식 감지 호환인가? (80자 이하, 한글 25% 미만)

### 4. 연결의 정직성

SA는 직접 영감(Boltzmann Machine, softmax), 구조적 유사(epsilon-greedy, 학습률 스케줄링), 독립 발전을 **명확히 구분**한다. 모든 것을 "영감을 받았다"로 뭉뚱그리지 않는다.

**체크포인트:**
- AI 연결이 다음 중 하나로 분류되어 있는가?
  - 직접 영감 (수식/메커니즘이 실제로 이식됨)
  - 구조적 유사 (같은 직관을 독립적으로 공유)
  - 독립 발전 (이름만 유사)
- 과장된 인과 관계 주장이 없는가?
- connectionType(front-matter)과 본문 내용이 일치하는가?

### 5. 시각적 스캐너빌리티

모바일 화면에서 스크롤하며 읽는 사용자가 **각 섹션의 핵심을 시각적으로 빠르게 파악**할 수 있어야 한다.

**체크포인트:**
- `##` 제목만 훑어도 전체 흐름이 파악되는가?
- 긴 문단(6줄+)이 없는가? (3-4문장 단위로 분리)
- 불릿/번호 리스트가 나열형 정보에 적절히 사용되었는가?
- 용어 정리가 5-10개로 핵심만 담고 있는가?
- **콘텐츠 총 길이가 KO 기준 3,500~5,000자인가?** (이 범위를 초과하면 시각적 피로)

## 검토 프로세스: 에이전트 배분

### Phase 1: 학문적 정확성 + 교육 효과 (ai-interdisciplinary-scholar)

ai-interdisciplinary-scholar 에이전트에게 다음을 검토하게 한다:

```
다음 학문원리 콘텐츠를 검토해줘:
[콘텐츠 전문]

검토 기준:
1. 원래 학문(물리/생물/수학 등)의 원리가 정확하게 설명되어 있는가?
2. 원래 학문의 맥락에서 핵심 인사이트가 충분히 전달되는가?
3. AI와의 연결이 정직한가? (직접 영감 vs 구조적 유사 vs 독립 발전 구분)
4. 역사적 맥락(핵심 인물, 연도, 원래 목적)이 정확한가?
5. 누락된 핵심 개념이 있는가?
6. 잘못된 연결이나 과장된 인과 관계가 있는가?

Gold Standard(SA) 대비 이 원리의 교육적 효과를 A/B/C로 평가하고,
구체적 개선 제안을 제시해줘.
```

### Phase 2: 비유/시각화 개선 (prompt-engineer)

prompt-engineer 에이전트에게 다음을 검토하게 한다:

```
다음 학문원리 콘텐츠의 '가르치는 능력'을 검토해줘:
[콘텐츠 전문]

Gold Standard 비유 예시 (SA):
"목적 함수의 값을 높이로 표현한 울퉁불퉁한 산악 지형에서, 가장 깊은 골짜기를 찾는 것이 목표다."

검토 기준:
1. 내러티브 아크: 원래 학문 → 인사이트 → 역사 → AI 연결 흐름이 자연스러운가?
2. 공간적 비유: 추상적 개념을 시각적으로 그릴 수 있게 하는 비유가 있는가?
3. 수식 해체: 수식이 있으면 변수별 의미 + 직관적 해석이 있는가?
4. 콘텐츠 길이: 교육에 필요한 최소한의 분량인가? 불필요한 반복이나 과잉 설명이 없는가?
5. 시각적 스캐너빌리티: 모바일에서 스크롤하며 빠르게 핵심 파악이 가능한가?

각 항목에 A/B/C 등급 + 구체적 개선 제안을 제시해줘.
특히, 비유가 부족한 섹션에 대해 새로운 비유를 제안해줘.
```

### Phase 3: 시뮬레이션 필요성 평가 (simulation-expert + app-developer)

시뮬레이션이 없는 원리(SEED_TO_SIM에 없는 원리)에 대해 **simulation-expert** 에이전트로 교육적 임팩트를 평가하고, 필요 시 **app-developer**로 기술 구현 가능성을 보조 확인한다.

**Step 1: 교육적 임팩트 평가 (simulation-expert)**

```
다음 학문원리에 인터랙티브 시뮬레이션이 필요한지 평가해줘:
원리: {principle_name}
콘텐츠 요약: {one_line_summary}

현재 시뮬레이션이 있는 원리: SA (에너지 지형 탐색), GD (등고선 + 3종 옵티마이저),
Swarm (군집 이동 시각화), Bayesian (사전/사후 분포 업데이트)

평가 기준:
1. 시뮬레이션 없이 텍스트만으로 핵심 메커니즘을 이해할 수 있는가?
2. 파라미터 조절이 결과에 미치는 영향을 체험하는 것이 교육적 가치가 높은가?
3. 시각적 피드백(그래프, 애니메이션)이 텍스트 설명보다 효과적인 개념이 있는가?
4. 시뮬레이션 추가 시 사용자 이해도 향상 폭을 예측해줘 (what-if 시나리오).
5. 기존 시뮬레이션(SA, GD 등)과 비교해 구현 대비 교육 효과 ROI를 판단해줘.

결론: ESSENTIAL(필수) / BENEFICIAL(도움됨) / OPTIONAL(텍스트 충분) 중 하나로 판정.
ESSENTIAL/BENEFICIAL이면 어떤 시각화가 효과적일지 구체적으로 제안해줘.
```

**Step 2 (ESSENTIAL/BENEFICIAL만): 기술 구현 가능성 확인 (app-developer)**

simulation-expert가 ESSENTIAL/BENEFICIAL로 판정한 경우에만 app-developer에게 React Native 환경에서의 구현 가능성과 기술적 제약을 확인한다.

> **주의**: 시뮬레이션 개발은 이 스킬의 범위가 아니다. 필요성 평가와 제안만 수행한다.

### Phase 4: 렌더링 호환성 (자동)

직접 수행한다 (에이전트 불필요):

- SnapsContentRenderer 11가지 블록 타입 호환성
- 미지원 마크다운 (취소선, 이탤릭, 링크, 테이블, 이미지, HTML, LaTeX, 중첩 리스트)
- 수식 감지 조건: 80자 이하 + 한글 25% 미만 + 수학 패턴
- definition 오인식 가능성 (40자 초과 term, `- ` 로 시작하는 일반 불릿)
- 연속 definition → definition_group 자동 병합

### Phase 5: 수정 (B 이하만)

수정이 필요하면 에이전트를 활용한다:

| 수정 유형 | 에이전트 |
|-----------|---------|
| 학문적 내용 (원리 설명, AI 연결, 역사) | ai-interdisciplinary-scholar |
| 비유 추가/개선, 내러티브 재구성, 수식 해체 | prompt-engineer |
| 렌더링 호환성 수정 | 직접 수행 |

**수정 원칙:**
- SA를 참조하되 복사 금지
- A등급 차원은 보존 (좋은 부분을 망가뜨리지 않기)
- 콘텐츠 길이 KO 3,500~5,000자 (SA 수준 +-500자)
- EN 동기화 필수 (KO 수정 시 EN도 같이 수정)
- 렌더러 호환 유지
- **opt_simulated_annealing.md는 절대 수정 금지** -- Gold Standard 참조 전용
- 수정 시 KO + EN 동시 수정. EN은 번역체가 아닌 자연스러운 영문

## 등급 판정

### 5개 차원 + 가중치

| 차원 | 가중치 | A (SA 수준) | C (미달) |
|------|--------|------------|---------|
| 내러티브 아크 | 30% | 원래 학문 독립 이해 → 인사이트 → AI 자연 전환 | AI부터 시작하거나 원래 학문 피상적 |
| 공간적 비유 | 25% | 핵심 개념을 머릿속에 그릴 수 있는 비유 2+ | 비유 없이 추상적 설명만 |
| 연결의 정직성 | 20% | 직접/구조적/독립 정확히 구분 | 모든 연결을 "영감" 으로 뭉뚱그림 |
| 수식/메커니즘 해체 | 15% | 변수별 의미 + 직관적 해석 + 물리적 유래 | 수식만 제시하거나 수식 없이 말로만 |
| 시각적 스캐너빌리티 | 10% | 제목만 훑어도 흐름 파악, 적절한 분량 | 긴 문단, 벽 같은 텍스트 |

**종합**: A(85%+) / B(65-84%) / C(<65%)

## 수정 후 검증 체크리스트

수정 완료 후 반드시 확인:

1. **front-matter 보존**: difficulty, connectionType, keywords, keywords_en 그대로인가?
2. **`---EN---` 구분자**: KO/EN 분리 정상인가?
3. **길이 범위**: KO 3,500~5,000자, EN은 KO의 70~130%
4. **렌더러 호환**: 미지원 마크다운 없는가?
5. **`_load_curated_content()` 호환**: `scripts/agents/principle_team.py`의 파싱 로직과 호환되는가?
6. **principle_seeds.py 교차 확인**: seed 메타데이터와 콘텐츠 정보 일치하는가?
7. **state.json 업데이트**: 수정 후 등급과 날짜를 `.claude/lab-review-state.json`에 반영했는가?

qa-pipeline-tester 에이전트로 수정 전/후 KO-EN 일관성을 검증할 수 있다.

## 주의사항

- Phase 1 + Phase 2는 병렬 실행, Phase 3은 시뮬레이션 미보유 원리만 (simulation-expert → 판정 후 app-developer 보조)
- 일괄 시 super_category별로 분할 실행 (공학 → 자연과학 → 형식과학 → 응용과학)
- 수정은 기존 파일 Edit, 새 파일 생성 금지
- 수정 후 `_load_curated_content()` 호환성 확인 (KO/EN 분리, front-matter 보존)
- 종합 A 등급은 리포트만 제공, 수정하지 않음
- `principle-qa`와 중복 실행 불필요 — 이 스킬이 교육 효과를, `principle-qa`가 구조/형식을 담당
- scan 모드로 먼저 전체 현황 파악 후 개별 심층 검토를 권장
- state.json은 `.gitignore` 대상 — 검토 상태는 로컬 전용
