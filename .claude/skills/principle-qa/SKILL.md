# Principle QA — 학문스낵 curated 콘텐츠 품질 검토 및 수정

학문스낵 curated 원리 콘텐츠(`scripts/curated_principles/*.md`)의 품질을 Gold Standard인 **Simulated Annealing**(opt_simulated_annealing.md)과 비교하여 검토하고, 기준 미달 콘텐츠를 수정합니다.

## 사용법

- `/principle-qa` — 전체 45개 원리 중 미검토 또는 이전 검토에서 B 이하 등급을 받은 원리를 자동 선택하여 검토
- `/principle-qa opt_simulated_annealing` — 특정 원리 검토 (확장자 생략 가능)
- `/principle-qa all` — 전체 45개 원리 일괄 검토 (리포트만, 수정은 B 이하만)
- `/principle-qa fix neuro_attention_mechanism` — 특정 원리를 검토 없이 바로 수정 (이전 검토 결과 기반)

## Phase 1: Gold Standard 기준 수립

**참조 파일:** `scripts/curated_principles/opt_simulated_annealing.md` (4,542자 KO, ~9분 읽기)

SA에서 추출한 품질 기준선:

### 1.1 구조 기준

| 요소 | SA 기준 | 필수 여부 |
|------|--------|----------|
| Front-matter | difficulty, connectionType, keywords, keywords_en | 필수 |
| 제목 줄 | "English Name - 한줄 정의" (콘텐츠 첫 줄) | 필수 |
| 학문 원리 섹션 | 원래 학문 분야의 원리를 독립적으로 설명 (AI 언급 없이) | 필수 |
| 알고리즘 번역 섹션 | 학문 원리 → AI 알고리즘으로의 매핑 (대응 관계 명시) | 필수 |
| 핵심 메커니즘 | 수학적/논리적 핵심을 구체적으로 설명 (수식 포함 가능) | 필수 |
| 현대 AI 연결 | 해당 원리가 현대 AI 기법에 어떻게 살아있는지 | 필수 |
| 한계/약점 | 솔직한 한계 평가 (만능이 아님을 명시) | 필수 |
| 용어 정리 | 전문 용어 5-10개 정의 (term - definition 형식) | 필수 |
| ---EN--- 섹션 | 전체 영문 버전 (번역체 아닌 자연스러운 영어) | 필수 |

### 1.3 콘텐츠 깊이 기준 (★ 핵심)

| 차원 | A 등급 (SA 수준) | B 등급 (양호) | C 등급 (미달) |
|------|-----------------|-------------|-------------|
| **학문 원리 설명** | 원래 학문의 맥락에서 독립적으로 이해 가능. 물리적/생물학적/수학적 직관이 명확 | 원리 자체는 설명되지만 AI 연결 전 독립 이해가 부족 | AI와 바로 연결하여 원래 학문의 깊이 부족 |
| **역사적 맥락** | 핵심 인물 + 연도 + 원래 목적 + 후속 발전 (SA: Metropolis 1953 → Kirkpatrick 1983) | 주요 인물과 연도는 있지만 발전 계보가 불완전 | 역사적 맥락 거의 없음 |
| **수식/수학적 표현** | 핵심 수식 + 각 변수 설명 + 물리적 의미 (SA: P = e^(-dE/T), 각 변수 매핑) | 수식은 있지만 변수 설명이 부족하거나 물리적 의미 누락 | 수식 없이 말로만 설명 (수학적 분야인데 수식 없으면 C) |
| **AI 연결의 구체성** | 대응 관계를 항목별로 명시 + 직접 영감 vs 구조적 유사성 구분 (SA: 5개 대응 관계 명시) | AI 연결은 있지만 대응 관계가 모호하거나 구분 없음 | "AI에 활용된다" 수준의 추상적 언급 |
| **공간적 비유/직관** | 복잡한 개념을 공간적 비유로 시각화 (SA: 산악 지형 비유, 숲속 길 비유) | 비유가 1-2개 있지만 핵심 개념에 대한 것이 아님 | 비유 없이 추상적 설명만 |
| **한계 평가** | 구체적 한계 3-4개 + 왜 한계인지 설명 (SA: 고차원 약함, 그래디언트 미활용 등) | 한계를 언급하지만 1-2개이거나 피상적 | 한계 섹션 없음 또는 "한계가 있다" 수준 |
| **연결의 정직성** | 직접 영감 / 구조적 유사 / 독립 발전을 정확히 구분 | 구분 없이 모두 "영감" 취급 | 과장된 인과 관계 (X가 Y를 낳았다 식 오류) |

### 1.4 명확성 기준

| 항목 | 기준 | 위반 예시 |
|------|------|----------|
| **용어 선설명** | 전문 용어는 처음 등장 시 풀어서 설명한 뒤 사용 | 갑자기 "에르고딕성"이 등장 → 설명 없이 넘어감 |
| **문장 분리** | 복합 개념은 2-3문장으로 나누어 설명 | 하나의 문장에 3개 이상의 개념이 나열됨 |
| **구체적 사례** | 추상적 원리마다 구체적 적용 사례 1개 이상 | "다양한 분야에 활용된다" (구체적 분야/문제 미제시) |
| **인과 계보 정확성** | 영감 관계를 과장하지 않음 (독립 발전은 독립 발전으로) | "X에서 영감받아 Y가 만들어졌다" (실제로는 독립 발전) |
| **수식 중간 단계** | 최종 수식만 나열하지 않고 도출 과정 또는 각 항의 의미 설명 | P(theta|D) = P(D|theta) * P(theta) / P(D) 만 나열 |
| **대상 독자 적합성** | AI 관심 일반인~초중급이 이해 가능한 수준 | 대학원 교재 수준의 전문 서술 (너무 어려움) 또는 초등학생 수준 (너무 쉬움) |

## Phase 2: 품질 검토 (8개 평가 차원)

대상 파일을 읽고 아래 8개 차원에서 평가합니다. **ai-interdisciplinary-scholar 에이전트를 반드시 사용하여 학문적 정확성을 검증합니다.**

### 2.1 구조 검사 (자동)

파일을 읽고 다음을 자동 확인:

1. **Front-matter 완결성**: difficulty, connectionType, keywords, keywords_en 존재 여부
2. **제목 줄 형식**: 첫 줄이 "English Name - 한줄 정의" 패턴인지
3. **섹션 구조**: `##` 섹션 수, 필수 섹션(원리/매핑/연결/한계/용어) 존재 여부
4. **---EN--- 구분자**: 존재 여부 + EN 본문 존재 여부
5. **길이**: KO 본문 글자 수 (4,242~4,842자 범위)
6. **EN 비율**: EN 글자 수가 KO의 70~130% 범위
7. **용어 정리**: "term - definition" 패턴 항목 수 (최소 4개)
8. **렌더러 호환성**: 미지원 마크다운 사용 여부 (테이블, 링크, 이미지, HTML, LaTeX)

### 2.2 학문적 정확성 검사 (ai-interdisciplinary-scholar)

**반드시 ai-interdisciplinary-scholar 에이전트를 사용.** 다음을 검증:

1. **원리 설명의 정확성**: 원래 학문 분야에서의 원리가 정확히 서술되었는지
   - 핵심 개념이 왜곡 없이 전달되는지
   - 역사적 사실(인물, 연도, 맥락)이 정확한지
   - 수식/수학적 표현이 올바른지
2. **AI 매핑의 정확성**: 학문 원리와 AI 기법의 대응 관계가 정확한지
   - 직접 영감 vs 구조적 유사성 vs 독립 발전이 올바르게 구분되었는지
   - 인과 관계가 과장되지 않았는지
   - 매핑되는 AI 기법의 설명이 정확한지
3. **누락된 핵심 개념**: 해당 원리를 이해하는 데 빠져서는 안 되는 개념이 빠졌는지
4. **잘못된 연결**: AI와의 연결에서 사실과 다른 주장이 있는지

**ai-interdisciplinary-scholar 호출 프롬프트 템플릿:**

```
다음 학문스낵 콘텐츠의 학문적 정확성을 검증해주세요.

원리: {principle_name}
분야: {discipline}
AI 연결 유형: {connectionType}

[콘텐츠 전문 붙여넣기]

검증 항목:
1. 원래 학문 분야에서의 원리 설명이 정확한가? (핵심 개념, 역사적 사실, 수식)
2. AI 기법과의 대응 관계가 정확한가? (직접 영감 vs 구조적 유사성 vs 독립 발전)
3. 인과 관계가 과장되었거나 잘못된 곳이 있는가?
4. 해당 원리를 설명할 때 빠져서는 안 되는 핵심 개념이 누락되었는가?
5. 대상 독자(AI 관심 일반인~초중급)에게 적절한 난이도인가?

각 항목에 대해 [정확/부분적 오류/오류] 판정과 구체적 근거를 제시해주세요.
```

### 2.6 원리 간 중복 검사

**45개 원리 간 콘텐츠 중복 검사.** 다음 유형의 중복을 확인:

1. **AI 연결 중복**: 같은 AI 기법을 여러 원리에서 핵심 연결로 사용
   - 허용: 같은 AI 기법이 다른 관점에서 언급 (e.g., 볼츠만 머신이 SA와 Boltzmann Distribution에서 다른 측면 설명)
   - 문제: 두 원리에서 동일한 AI 기법을 거의 같은 방식으로 설명
2. **설명 패턴 중복**: 비유/사례가 다른 원리와 겹침
3. **용어 정리 중복**: 같은 용어가 여러 원리에서 정의됨 (허용되나, 정의가 일관되어야 함)

### 3.2 종합 등급

- **A (Gold Standard)**: 가중 평균 90%+ (SA와 동등한 수준)
- **B (양호)**: 가중 평균 70-89% (출판 가능하나 개선 여지)
- **C (미달)**: 가중 평균 50-69% (수정 필요)
- **D (부적격)**: 가중 평균 <50% (전면 재작성 필요)

## Phase 4: 수정 (B 이하 등급만)

종합 등급이 B 이하인 원리에 대해 수정을 수행합니다.

### 4.1 에이전트 선택

| 수정 유형 | 에이전트 | 역할 |
|-----------|---------|------|
| 학문적 오류 수정, 누락 개념 보충, AI 매핑 정확도 개선 | ai-interdisciplinary-scholar | 학문적 내용 작성/수정 |
| 명확성 개선, 비유 추가, 난이도 조정, 문장 분리 | ai-interdisciplinary-scholar | 교육적 서술 개선 |
| 구조 재배치, 섹션 추가/삭제, 길이 조정 | (직접 수행) | 편집 작업 |
| 바이링구얼 품질 개선, EN 자연스러움 | ai-interdisciplinary-scholar | 영어 문체 교정 |

### 4.2 수정 원칙

1. **SA를 참조하되 복사하지 않는다** — SA의 구조와 깊이를 모방하되, 해당 원리 고유의 서술 방식을 유지
2. **기존에 잘 된 부분은 건드리지 않는다** — C 등급 차원만 수정, A 등급 차원은 보존
3. **수정 범위를 최소화한다** — 전면 재작성보다 삽입/수정/삭제로 해결
4. **렌더러 호환성을 유지한다** — 수정 후에도 SnapsContentRenderer가 올바르게 파싱하는지 확인
5. **길이 범위를 준수한다** — 수정으로 인해 4,242~4,842자 범위를 벗어나지 않도록
6. **EN 섹션도 동기화한다** — KO 수정 시 EN도 함께 수정

### 4.3 수정 절차

1. C 등급 차원을 우선 수정 (심각도 순: 학문적 정확성 > 콘텐츠 깊이 > 명확성 > 기타)
2. B 등급 차원 중 쉽게 개선 가능한 것 수정
3. 수정 후 길이 재측정 (4,242~4,842자 확인)
4. 수정 후 렌더러 호환성 재확인

### 4.4 수정 시 ai-interdisciplinary-scholar 호출

학문적 내용 수정이 필요한 경우, ai-interdisciplinary-scholar 에이전트에게 다음 정보를 제공:

```
다음 학문스낵 콘텐츠를 수정해주세요.

원리: {principle_name}
분야: {discipline}
현재 등급: {grade}

[수정이 필요한 부분 + 현재 텍스트]

수정 요청:
1. {구체적 수정 사항 1}: {현재 문제점} → {기대하는 방향}
2. {구체적 수정 사항 2}: {현재 문제점} → {기대하는 방향}
...

제약 조건:
- 대상 독자: AI 관심 일반인~초중급
- 수식은 LaTeX가 아닌 평문 표기 (dE = E(x') - E(x) 형태)
- 전문 용어는 처음 등장 시 풀어서 설명
- 인과 관계 과장 금지 (직접 영감 vs 구조적 유사성 vs 독립 발전 구분)
- 공간적 비유로 직관 제공
- KO 4,242~4,842자 범위 유지
```

**수정은 반드시 병렬 에이전트로 실행:**
- KO 내용 수정 + EN 동기화를 별도 에이전트로 나누지 말 것 (같은 에이전트가 KO+EN 동시 수정)
- 여러 원리를 수정할 때는 원리별로 별도 에이전트 병렬 실행 가능
